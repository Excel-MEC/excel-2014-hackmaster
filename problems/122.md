  <p>The most naive way of computing n15 requires fourteen multiplications:</p>  <p style='margin-left:100px;'>n &times; n &times; ... &times; n = n15</p>  <p>But using a &quot;binary&quot; method you can compute it in six multiplications:</p>  <p style='margin-left:100px;'>n &times; n = n2<br />  n2 &times; n2 = n4<br />  n4 &times; n4 = n8<br />  n8 &times; n4 = n12<br />  n12 &times; n2 = n14<br />  n14 &times; n = n15</p>  <p>However it is yet possible to compute it in only five multiplications:</p>  <p style='margin-left:100px;'>n &times; n = n2<br />  n2 &times; n = n3<br />  n3 &times; n3 = n6<br />  n6 &times; n6 = n12<br />  n12 &times; n3 = n15</p>  <p>We shall define m(k) to be the minimum number of multiplications to compute nk; for example m(15) = 5.</p>  <p>For 1 &le; k &le; 200, find <img src='images/symbol_sum.gif' width='11' height='14' alt='&sum;' border='0' style='vertical-align:middle;' /> m(k).</p>
